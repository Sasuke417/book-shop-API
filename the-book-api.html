1. Django Project - E2 - The Book Shop

The Book Shop is an online e-commerce platform for purchasing books. You can browse through the book collection, add books to a cart, and purchase with an order confirmation.

 

The Book Shop is built using Django REST Framework, and implements API for all the features. The features include:

  • Inventory - Defines the collection of books by genre, author, and publisher.
  • User - Users who can create login accounts and use it to order books.
  • Order Management - Enables to add books to a cart, provide address, and place orders.


1. Inventory feature

The basis of the bookshop is inventory collection. Develop the API feature to list and detail:

  • Genres
  • Authors
  • Publishers
  • Books

A fixture seeddata.json is provided to setup inventory data needs. Use the command:

$ python manage.py loaddata the_book_shop_api/inventory/fixtures/seeddata.json
The API should not allow users to POST, EDIT, DELETE data from the inventory.

 

2. User feature

The User API extends Django's default django.contrib.auth.models.User, and enables to:

  • Create a login account.
  • Login to an account with token validation.
  • Display user details.

 

3. Order Management feature

Build a set of API to handle orders after user login.

  • Maintain an order (create, read, update, and delete).
  • Add, updated, delete, and review items in an order.
  • Add a physical address to an order, and review the final order.
  • Place the order.
  • Send a summary of the order by email to the user.

Note: Assume payment is done automatically.

 

The following are provided:

 

  • Database sqllite3 for the tables and migration.
  • Inventory as a fixture-based input in the_book_shop_api/inventory/fixtures/seeddata.json.
  • Project structure which is built using cookie-cutter.
  • All the necessary import functions.
  • Tests to validate the code.

 

Complete the project so that it passes all test cases.
